# 用例格式优化方案 · 仅修改 Agent 定义与 Task 交互

> **目标**：解决单元格换行无效、用例概述/前置条件/测试步骤之间的场景重复问题。  
> **原则**：仅修改 `config/agents.yaml` 中的 agent 定义与 task 描述，尽量不修改代码。

---

## 一、问题与根因

| 问题 | 根因 |
|------|------|
| 1. 单元格内 `<br>` 换行不生效 | Excel 单元格换行需使用换行符 `\n`，当前导出逻辑未将 `<br>` 转为 `\n`，导致 `<br>` 以原文显示 |
| 2. 用例概述已写明场景时，前置条件仍重复写场景 | Agent 未约束「用例概述与前置条件」的职责边界，导致冗余 |
| 3. 前置条件已有场景时，测试步骤仍重复写场景 | Agent 未约束「前置条件与测试步骤」的衔接，导致步骤冗余 |

---

## 二、优化方案（仅改 agents.yaml）

### 2.1 单元格换行：弃用 `<br>`，改用「同单元格内分号分隔」

**原因**：在不改代码的前提下，`<br>` 无法在 Excel 中呈现为换行。改用「同单元格内分号分隔」可避免换行需求，且可读性尚可。

**修改点**：

- **case_agent.backstory** 中「预期结果」撰写准则：
  - 原：`多条结果使用<br>1. ...的编号换行格式`
  - 新：`多条结果在同一单元格内使用「1. xxx；2. yyy」格式，分号分隔，禁止使用<br>或换行符（Excel 无法正确解析）`

- **review_agent.backstory** 中验收标准：
  - 原：`预期结果多条时使用<br>1. 2. 3.编号`
  - 新：`预期结果多条时使用「1. xxx；2. yyy」分号分隔格式，禁止<br>`

- **task3.description** 中生死红线：
  - 原：`单元格内部需要换行时，必须且只能使用 <br> 标签`
  - 新：`单元格内禁止换行，多条内容使用「1. xxx；2. yyy」分号分隔，禁止使用 <br> 或换行符`

- **task4.description** 中字段规范：
  - 原：`多条结果是否使用了<br>换行？`
  - 新：`多条结果是否使用了分号分隔格式？`

---

### 2.2 用例概述与前置条件：避免场景重复

**原则**：用例概述负责「场景/对象定位」，前置条件仅补充「该场景下未涵盖的其它必要条件」。

**修改点**：

- **case_agent.backstory** 中「前置条件」准则：
  - 新增：`若用例概述已写明场景（如「直播间-礼物面板-余额不足态」），前置条件仅补充该场景下未涵盖的其它必要条件（如账号、网络、数据状态），禁止重复描述场景本身`

- **organize_agent.backstory** 中拆解原则（可选，用于 2 号输出更清晰）：
  - 新增：`用例概述采用「大模块-小组件-具体元素/状态」，前置条件由 3 号根据概述补充，不重复场景`

---

### 2.3 前置条件与测试步骤：避免步骤重复场景

**原则**：前置条件已描述的场景，测试步骤从「达成该场景后的第一个操作动作」开始，不再重复场景。

**修改点**：

- **case_agent.backstory** 中「测试步骤」准则：
  - 新增：`若前置条件已写明场景或初始状态，测试步骤从达成该场景后的第一个操作动作开始，禁止在步骤中重复描述「进入xxx页面」「确保xxx状态」等前置内容`

- **task3.description** 中任务执行说明：
  - 新增：`前置条件与测试步骤禁止重复：前置条件已有的场景/状态，测试步骤中不再复述，直接从第一个操作动作开始`

## 四、可选：最小代码改动（若需保留换行）

若产品坚持在 Excel 中保留单元格内换行，可在 `crew_test.py` 的 `_sanitize_cell_for_excel` 中增加一行：

```python
s = re.sub(r"<br\s*/?>", "\n", s, flags=re.I)
```

这样 Agent 继续输出 `<br>` 即可在 Excel 中正确换行。但本方案优先采用「仅改 Agent」的方式，避免动代码。

## 五、验收预期

| 验收项 | 预期 |
|--------|------|
| 预期结果多条 | 单元格内为「1. xxx；2. yyy」格式，无 `<br>`，Excel 中正常显示 |
| 用例概述含场景 | 前置条件不重复场景，仅补充账号、网络、数据等必要条件 |
| 前置条件含场景 | 测试步骤从第一个操作动作开始，不重复「进入xxx」「确保xxx」 |
| 审查通过 | review_agent 按新规范校验，不要求 `<br>`，检查分号分隔与无重复 |
