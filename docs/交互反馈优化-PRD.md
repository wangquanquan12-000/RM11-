# 交互反馈优化 · PRD

> **文档用途**：供开发优化「用例工坊」整体交互体验，统一按钮点击后的响应、保存/更新成功的提示、以及「无变更保存」时的反馈。
>
> **创建日期**：2025-03-02

---

## 一、优化目标

1. **按钮点击后**：有明确的加载态或结果反馈，避免用户重复点击或误判为无响应。
2. **保存/更新成功**：统一使用 `st.success` 展示明确成功文案。
3. **无变更保存**：当用户点击保存但内容与当前已持久化数据完全一致时，给出「内容未变更，无需保存」类提示，而非执行写入。

---

## 二、涉及场景清单

| 模块 | 操作 | 当前反馈 | 需优化点 |
|------|------|----------|----------|
| 编辑 Agent | 保存配置到 agents.yaml | 成功：`st.success("已保存，下次生成用例将使用新配置")` | 无变更时提示「内容未变更，无需保存」 |
| 项目记忆 | 保存摘要 | 成功：`st.success("已保存")` | 无变更时提示「内容未变更，无需保存」 |
| 项目记忆 | 从本次运行追加 | 成功：`st.success("已追加到项目记忆")` | 无可追加内容时已有 `st.info`；可补充「无可追加内容」 |
| 项目记忆 | 刷新知识库 | 成功：`st.success("知识库已更新")`；失败：`st.error` | 执行中需 spinner，按钮禁用 |
| 设置 | 保存 Token/Key/模型 | 成功：`st.success("已保存到本地（xxx）")` | 无变更时提示「内容未变更，无需保存」 |
| 生成用例 | 点击生成 | `st.progress` + 禁用 | 保持现有逻辑，确保错误时有 `st.error` |
| 需求风险分析 | 生成风险报告 | spinner + 禁用 | 保持，确保成功/失败提示清晰 |
| 导入（文件夹/单文档/全回归） | 点击导入 | 拉取中 spinner、成功/失败 | 保持，确保成功文案统一 |

## 三、详细规范

### 3.1 保存类操作：无变更时的反馈

**规则**：在执行写入前，比较当前表单/输入与已持久化数据；若完全一致，则**不执行写入**，并提示「内容未变更，无需保存」。

| 操作 | 比较对象 | 提示文案（配置 key） |
|------|----------|----------------------|
| 保存 Agent 配置 | 当前 agents/tasks 表单 vs 上次保存的 hash | `save_no_change: "内容未变更，无需保存"` |
| 保存项目记忆摘要 | `new_mem` vs `load_project_memory()` | `save_no_change: "内容未变更，无需保存"` |
| 保存设置（Token/Key/模型） | 当前输入 vs `get_credentials()` 或 defaults | `save_no_change: "内容未变更，无需保存"` |

- 无变更时使用 `st.info` 或 `st.warning`，建议 `st.info`。
- 文案统一放入 `config/ui_texts.yaml`，如 `common.save_no_change`。

### 3.2 保存/更新成功后的提示

**规则**：所有成功写入必须展示 `st.success`，文案具体且可区分。

| 操作 | 成功文案（配置 key） | 示例 |
|------|----------------------|------|
| 保存 Agent 配置 | `agents_tab.save_success` | 「已保存，下次生成用例将使用新配置」 |
| 保存项目记忆摘要 | `memory_tab.save_summary_success` | 「项目记忆摘要已保存」 |
| 从本次运行追加 | `memory_tab.append_success` | 「已追加到项目记忆」 |
| 刷新知识库 | `memory_tab.knowledge_refresh_success` | 「知识库已更新」 |
| 保存设置 | `run_tab.save_success` | 「已保存到本地（Keyring/JSON）」 |

- 所有文案从 `ui_texts.yaml` 读取，禁止硬编码。

### 3.3 按钮点击后的响应

| 类型 | 要求 |
|------|------|
| **异步操作**（拉取、生成、刷新知识库等） | 1）点击后立即 `st.spinner` 或 `st.progress`<br>2）对应按钮 `disabled=True` 防止重复点击<br>3）完成后 `st.rerun()` 或清除 loading 态 |
| **同步保存**（保存配置、保存摘要等） | 1）保存成功后 `st.success`<br>2）建议 `st.rerun()` 以刷新 dirty 等状态<br>3）无变更时 `st.info`，不 rerun |
| **失败** | 使用 `st.error`，文案包含可操作提示（如「请检查 Token」「请稍后重试」） |

### 3.4 「无可追加」类场景

| 场景 | 提示方式 | 文案示例 |
|------|----------|----------|
| 点击「从本次运行追加」但无上次运行数据 | `st.info` | 「请先在「生成用例」页运行一次」 |
| 可补充 | `st.info` | 「暂无新的运行记录可追加」 |

## 四、文案配置（ui_texts.yaml 新增）

在 `config/ui_texts.yaml` 中新增或补充：

```yaml
common:
  save_no_change: "内容未变更，无需保存"

agents_tab:
  save_success: "已保存，下次生成用例将使用新配置"   # 若已有可复用

memory_tab:
  save_summary_success: "项目记忆摘要已保存"
  append_success: "已追加到项目记忆"
  append_no_data: "暂无新的运行记录可追加"           # 可选
  knowledge_refresh_success: "知识库已更新"
  knowledge_refresh_fail: "更新失败：{err}"

run_tab:
  save_success: "已保存到本地"                       # 已有，保持
```

## 五、验收标准

| 编号 | 验收项 |
|------|--------|
| AC1 | 保存 Agent 配置时，若内容无变更，提示「内容未变更，无需保存」，不执行写入 |
| AC2 | 保存项目记忆摘要时，若内容无变更，提示「内容未变更，无需保存」，不执行写入 |
| AC3 | 保存设置（Token/Key/模型）时，若内容无变更，提示「内容未变更，无需保存」，不执行写入 |
| AC4 | 所有保存/更新成功场景均有明确的 `st.success` 提示 |
| AC5 | 异步操作（生成用例、刷新知识库、导入、风险分析等）执行中，对应按钮禁用并显示 loading |
| AC6 | 所有新增或修改的提示文案均从 `ui_texts.yaml` 读取，无硬编码 |
