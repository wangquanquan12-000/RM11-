## 二、安全与合规类

### 2.1 Excel/CSV 公式注入防御

| 项目 | 内容 |
|------|------|
| **问题简述** | AI 产出的测试用例在导出 Excel 时，若单元格内容以 `=`、`+`、`-`、`@` 开头，可能被 Excel 解析为公式，存在潜在安全与展示异常风险。 |
| **影响范围** | 所有导出 Excel 的用户；涉及对外交付场景的合规与数据安全。 |
| **建议方案** | 在写入 Excel 前，对上述开头字符的单元格自动追加单引号，使其以纯文本形式存储。 |
| **产品决策** | ☐ 通过：按规范执行  ☐ 暂缓  ☐ 不采纳 |
| **PM 意见** | |

### 2.2 线上部署的凭证管理（st.secrets）

| 项目 | 内容 |
|------|------|
| **问题简述** | 当前凭证（Gemini Key 等）来自环境变量或本地文件。若部署到 Streamlit Cloud 等云环境，规范建议使用 `st.secrets` 管理敏感信息。 |
| **影响范围** | 线上/云部署场景；涉及凭证泄露与合规要求。 |

### 3.1 UI 与业务逻辑分离（MVC）
