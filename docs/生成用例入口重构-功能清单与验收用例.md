# 生成用例入口重构 · 功能清单与验收用例（实现版）

> **文档用途**：约定当前「生成用例」入口与历史记录功能的产品行为，作为已经落地实现的说明文档，供后续迭代与测试参考。
>
> **最近更新**：2025-03-XX（完全去除 Quip 入口，仅保留上传/粘贴；引入历史记录）

---

## 一、功能清单（F1–F6）

### F1 完全移除 Quip 入口，仅保留「上传」「粘贴」两种输入方式

- **F1-1 主入口保持为「生成用例」**
  - 侧栏使用「生成用例」作为入口，主区仅提供两种输入方式。
- **F1-2 仅两种主输入方式**
  - 方式一：**上传文件**（支持 .md / .docx / .xlsx）。
  - 方式二：**粘贴文本**（直接粘贴需求文档内容，可选上传 .xlsx）。
  - 通过 `st.radio` 或 Tab 在「上传文件」「粘贴文本」之间切换，**不提供 Quip 链接输入**。
- **F1-3 Quip 入口已去除**
  - 页面上**不再**展示任何 Quip 链接输入框、Quip 相关提示文案或 Quip 导出配置。
  - 如未来确需恢复 Quip 支持，需作为独立需求重新立项，不在本版本范围内。

---

### F2 支持上传/粘贴两种输入方式

- **F2-1 上传文件**
  - 支持多文件上传：`.md` / `.docx`（需求文档）、`.xlsx`（既有测试用例）。
  - 校验规则：
    - 至少需 1 个需求文档（.md 或 .docx）。
    - .xlsx 可选，用作既有用例上下文。
  - 预览展示：
    - 文本类（.md / .docx）：文件名 + 前 200 字摘要。
    - 表格类（.xlsx）：文件名 + 行数。
  - 生成后**仅**展示**新测试用例表**（支持下载 Excel），不再展示「理解内容」「问题点」块。
- **F2-2 粘贴文本**
  - 提供大文本输入框，支持直接粘贴 PRD 内容。
  - 可选上传单个 `.xlsx` 作为既有用例上下文。
  - 与上传模式共用同一四 Agent 流水线与导出逻辑，结果**仅**为测试用例表，不产出理解内容或问题点。

---

### F3 历史记录命名规则

- **F3-1 标题格式**
  - 历史列表中每条记录的标题统一为：  
    `〈来源类型〉 · 〈截断后的需求标题〉`
  - 来源类型仅：`上传` / `粘贴`（无 Quip）。
- **F3-2 需求标题生成规则**
  - 上传模式：
    - 取首个 .md/.docx 文件名（去扩展名），若不存在则用「上传需求」。
  - 粘贴模式：
    - 取内容第一行或前 20 个字；若超出 20 字，加省略号「…」。
- **F3-3 时间与排序**
  - 历史记录保存生成时间 `YYYY-MM-DD HH:MM`。
  - 列表按时间倒序展示，最近一次生成在最上方。

---

### F4 历史记录：查询 / 删除 / 下载

- **F4-1 查询**
  - 在历史区域顶部提供搜索框：
    - 支持按关键字模糊匹配标题或来源类型（如输入「上传」「粘贴」）。
    - 默认展示最近 N 条记录（实现中为 20 条）。
- **F4-2 删除**
  - 每条历史记录右侧提供删除按钮（🗑️）。
  - 点击后进入二次确认：
    - 「确认删除」：从本地历史 JSON 中移除该条记录。
    - 「取消」：关闭确认区域，不做修改。
  - 删除操作**只影响历史记录本身**，不强制删除已生成的 Excel 文件。
- **F4-3 下载**
  - 若该次生成解析到有效测试用例表格：
    - 历史卡片上展示「📥 下载 Excel」按钮；
    - 下载文件名规则：  
      `测试用例_{标题简写}_{timestamp}.xlsx`  
      其中「标题简写」为安全处理后的需求标题片段，「timestamp」为不含冒号的时间戳。
  - 若无表格可导出：
    - 展示灰态文案「本次无可下载表格」。

### F6 跨 Tab 状态保持（缓存机制）

- **F6-1 主工作台 Tab 之间的状态保持**
  - 在同一浏览器会话内，用户在各个主工作台 Tab 中已填写的表单内容（如「生成用例」「项目记忆」「编辑 Agent」「设置」等），在 Tab 之间来回切换时必须保持不丢失。
  - 例如：
    - 先在「生成用例」页上传了需求文档 / 粘贴了文本但尚未点击生成；
    - 临时切换到「项目记忆」或「设置」页查看或修改配置；
    - 再切回「生成用例」时，先前上传/粘贴的内容仍然可见且可继续使用。
- **F6-2 页面内子 Tab 之间的状态保持**
  - 在同一页面内部（如「生成用例」中的「上传文档」「粘贴内容」切换），各子 Tab 中已输入的内容在切换时同样需要被保留：
    - 在「上传文档」Tab 中选定的文件，在临时切到「粘贴内容」Tab 后，再切回时仍然存在；
    - 在「粘贴内容」Tab 中输入的长文本，在切换到其他 Tab 再切回时仍然保留。
  - 仅当用户主动执行「重置/清空」动作时，相关内容才会被清除。
- **F6-3 刷新边界**
  - 浏览器整页刷新（F5 或重新打开链接）后，不强制要求继续保留之前的输入状态，本机制主要保证**同一会话内的 Tab 切换不丢内容**。


- **F5-1 运行状态**
  - 生成按钮在运行期间禁用，文案变为「运行中…」。
  - 上传、粘贴两种模式在各自的运行状态下互不影响。
- **F5-2 配置状态提示**
  - 在需求来源切换控件下方展示当前配置状态：
    - Token/Key 已配置：展示「Token/Key 已配置」类提示。
    - 未配置：展示「请在『设置』中配置 Token 与 Gemini API Key」。
- **F5-3 空状态**
  - 当历史记录为空时：
    - 历史区域展示空状态文案：「暂无生成记录，上传或粘贴需求后开始生成」。
- **F5-4 错误提示**
  - 错误场景（如未上传需求文档、粘贴为空、模型未配置、流水线异常）：
    - 使用明显的 `st.error` 或警告卡片展示错误原因。
    - 提供「重试」按钮在必要时重新触发生成。

## 二、验收用例（TC1–TC9）

### TC1 首次进入主入口布局

- **前置条件**：首次打开「生成用例」页，无历史记录。
- **步骤**：
  1. 打开应用首页，进入「生成用例」。
- **预期**：
  - 页面顶部仅为「上传文件」「粘贴文本」两种方式，**无 Quip 链接输入**。
  - 历史区显示空状态，引导「上传或粘贴需求后开始生成」。

---

### TC2 上传单个 .md 文件生成

- **前置条件**：Gemini Key 已配置；无历史记录要求。
- **步骤**：
  1. 选择「上传文件」方式。
  2. 上传单个 `a.md` 需求文档。
  3. 点击「开始生成」。
- **预期**：
  - 上传列表展示 `a.md` 文件名和前 200 字摘要。
  - 生成成功后历史记录新增 1 条：来源类型为「上传」，标题包含 `a`。
  - 历史卡片上可点击「下载 Excel」，文件名符合命名规则。

---

### TC3 上传 .docx 文件生成

- **前置条件**：Gemini Key 已配置；准备一个包含数段文字的 `b.docx`。
- **步骤**：
  1. 选择「上传文件」方式。
  2. 上传单个 `b.docx` 需求文档。
  3. 点击「开始生成」。
- **预期**：
  - 上传列表展示 `b.docx` 文件名与从文档提取的前 200 字摘要。
  - 生成成功后历史记录新增 1 条：来源类型为「上传」，标题包含 `b`。
  - Excel 下载按钮可用，内容来自四 Agent 流水线。

---

### TC4 粘贴文本生成

- **前置条件**：Gemini Key 已配置。
- **步骤**：
  1. 切换到「粘贴文本」方式。
  2. 在文本框粘贴一段 PRD 内容。
  3. 点击「开始生成」。
- **预期**：
  - 若未上传任何文件，仅粘贴文本也可以生成。
  - 历史记录新增 1 条：来源类型为「粘贴」，标题取粘贴内容首行或前 20 字。
  - 可从历史记录下载 Excel。

---

### TC5 上传多文件（需求+既有用例）

- **前置条件**：Gemini Key 已配置；准备 `c.md` 与 `cases.xlsx`。
- **步骤**：
  1. 选择「上传文件」方式。
  2. 同时上传 `c.md` 和 `cases.xlsx`。
  3. 点击「开始生成」。
- **预期**：
  - 上传列表中：`c.md` 显示为文本摘要，`cases.xlsx` 显示为表格行数。
  - 四 Agent 生成逻辑可正常使用既有用例作为上下文。
  - 历史记录标题来自 `c.md`；Excel 可下载。

---

### TC6 历史记录查询与排序

- **前置条件**：已有多条历史记录，包含不同来源类型与标题。
- **步骤**：
  1. 在历史记录上方输入关键字（如部分标题或来源类型）。
  2. 观察列表变化。
- **预期**：
  - 列表仅展示与关键字匹配的记录。
  - 记录按时间倒序排列。
  - 清空关键字后恢复完整列表。

---

### TC7 删除历史记录

- **前置条件**：至少有 1 条历史记录。
- **步骤**：
  1. 在某条历史卡片上点击删除按钮（🗑️）。
  2. 在确认区域选择「确认删除」。
- **预期**：
  - 该条历史记录从列表中消失。
  - 已下载的 Excel 文件不受影响。
  - 刷新页面后该记录不会重新出现。

---

### TC8 从历史记录下载 Excel

- **前置条件**：存在至少 1 条生成成功并有表格的历史记录。
- **步骤**：
  1. 在该条历史记录卡片中点击「下载 Excel」。
- **预期**：
  - 浏览器下载 `.xlsx` 文件。
  - 文件名符合规则：`测试用例_{标题简写}_{timestamp}.xlsx`。
  - 打开文件可见与当次生成一致的测试用例表格。

---


---

### TC10 主工作台 Tab 切换保持输入

- **前置条件**：浏览器处于同一会话，未刷新页面。
- **步骤**：
  1. 在「生成测试用例」页选择「粘贴内容」模式，粘贴一段较长 PRD 文本；
  2. 不点击生成，切换到「项目记忆」或「设置」Tab；
  3. 再切回「生成测试用例」Tab。
- **预期**：
  - 粘贴框中的文本内容完整保留；
  - 如此前在「上传文档」模式中已选择文件，该选择同样不会因 Tab 切换而丢失。

### TC11 页面内子 Tab 切换保持输入

- **前置条件**：浏览器处于同一会话，未刷新页面。
- **步骤**：
  1. 在「生成测试用例」页的「上传文档」Tab 中选择一个 `.docx` 文件；
  2. 切换到同页的「粘贴内容」Tab，输入一段文本；
  3. 再切回「上传文档」Tab。
- **预期**：
  - 之前上传的 `.docx` 文件仍然被选中并显示；
  - 切回「粘贴内容」Tab 时，之前输入的文本同样完整保留；
  - 仅在用户点击显式的「清空/重置」按钮时，上述内容才会被清除。

### TC9 错误场景与提示

- **前置条件**：
  - 场景 A：上传 .xlsx 但未上传任何需求文档（.md/.docx）。
  - 场景 B：上传不支持的文件类型。
  - 场景 C：Gemini Key 未配置。
- **步骤与预期**：
  - 场景 A：点击「开始生成」时，提示「至少需上传 1 个需求文档（.md 或 .docx）」。
  - 场景 B：不支持的类型在上传列表中被忽略，并通过提示说明支持的类型。
  - 场景 C：点击「开始生成」时，提示「请先在『设置』中配置 Gemini API Key」，不进入流水线。
