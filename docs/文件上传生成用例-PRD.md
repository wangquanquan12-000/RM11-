# 文件上传生成用例 · PRD

> **文档用途**：定义「上传 .md + .xlsx 多文件 → 分析理解 → 产出问题点与新用例表」的产品方案。替代已废弃的 `docs/本地文件模式-PRD.md`。
>
> **创建日期**：2025-03-02

---

## 一、需求概述

### 1.1 背景

- 原「本地文件模式」为目录扫描 + Word(.docx) + 保存到工作目录。
- 新方案：**仅上传入口**，支持 **.md（需求文档）+ .xlsx（既有测试用例）** 混合上传，产出 **理解内容 + 问题点 + 新用例表**，仅提供 **Excel 下载**。

### 1.2 目标

- **输入**：用户一次上传多个 .md 和 .xlsx 文件
- **处理**：解析 → 分析文档与既有用例 → 四 Agent 生成新用例
- **输出**：① 理解内容 ② 问题点 ③ 新测试用例表；支持 Excel 下载

---

## 二、方案变更对照

| 项目 | 原方案（已废弃） | 新方案 |
|------|------------------|--------|
| 输入方式 | 本地目录扫描 | **仅上传入口** |
| 导出方式 | 保存到工作目录、可配导出路径 | **仅 Excel 下载**，不配置导出路径 |
| 文件数量 | 单文件或目录内多选 | **支持多文件**（一次上传多个） |
| 文件类型 | .docx（Word） | **.md + .xlsx**（需求文档 + 既有测试用例） |
| 产出物 | 测试用例表 | **① 理解内容 ② 问题点 ③ 新测试用例表** |

## 三、核心流程

```
┌─────────────────────────────────────────────────────────────────────┐
│  1. 上传   │ 用户选择多个 .md 和 .xlsx（可混合）                        │
├─────────────────────────────────────────────────────────────────────┤
│  2. 解析   │ .md → 文本；.xlsx → 表格内容                              │
├─────────────────────────────────────────────────────────────────────┤
│  3. 分析   │ 分析文档与既有用例，产出「理解内容」与「问题点」              │
├─────────────────────────────────────────────────────────────────────┤
│  4. 生成   │ 经四 Agent 流水线生成新测试用例表                           │
├─────────────────────────────────────────────────────────────────────┤
│  5. 下载   │ 提供「下载 Excel」按钮                                    │
└─────────────────────────────────────────────────────────────────────┘
```

## 五、UI 与交互设计

### 5.1 上传入口

- **组件**：`st.file_uploader`，`accept_multiple_files=True`，`type=["md", "xlsx"]`（或 `accept_multiple_files` 与 `accept=".md,.xlsx"` 等，按 Streamlit 版本适配）
- **说明文案**：支持上传多个 .md（需求文档）和 .xlsx（既有测试用例），可混合选择
- **校验**：至少 1 个 .md；.xlsx 可选

### 5.2 解析与展示预览

- 上传成功后，展示简要预览：
  - .md：文件名 + 前 200 字摘要
  - .xlsx：文件名 + 行数/列数
- 用户点击「开始生成」后进入分析 + 生成流程

### 5.3 结果展示（三块布局）

| 区块 | 标题 | 内容 |
|------|------|------|
| 1 | 理解内容 | 对上传文档与既有用例的摘要与理解 |
| 2 | 问题点 | 风险、遗漏、矛盾、待确认项等 |
| 3 | 新测试用例表 | Markdown 表格展示 + Excel 下载按钮 |

### 5.4 Excel 下载

- **仅** 提供 `st.download_button`，将新用例表导出为 .xlsx
- 不增加：导出路径输入、Quip 导出、Google Sheets 导出等

## 六、数据流与接口

### 6.1 解析

```text
.md 文件 → read(encoding="utf-8") → 文本
.xlsx 文件 → openpyxl 读取 → 表格行列表（与 _parse_markdown_tables 或 parse_test_cases_file 结构一致）
```

### 6.2 合并与传入流水线

- **需求文档**：所有 .md 内容合并（可加分隔符）
- **既有用例**：所有 .xlsx 内容合并为文本（如 | 分隔），作为 project_context 或专门上下文
- **demand**：以 .md 合并内容为主；既有用例作为「已知测试用例」附加说明传入 Agent

### 6.3 分析阶段产出

- **理解内容**：可由单独 Agent 或 task1（文档分析师）扩展输出「理解摘要」
- **问题点**：可由 task1 的风险报告或单独分析产出
- **新用例表**：由 task1→task2→task3→task4 标准流水线产出

> 具体是「扩展现有四 Agent」还是「新增分析步骤」由实现文档细化。

## 七、安全与防御

| 项 | 说明 |
|------|------|
| 文件类型白名单 | 仅接受 .md、.xlsx；拒绝 .exe、.py、.bat 等 |
| 文件大小限制 | 建议单文件 &lt; 10MB，总上传 &lt; 50MB（可配置） |
| Excel 注入 | 解析后的内容写入 Excel 时，继续使用 `_sanitize_cell_for_excel` 防公式注入 |
| 不落盘 | 上传文件仅在内存处理，不写入用户不可控路径 |

## 十、实现要点（供开发参考）

### 10.1 上传组件

```python
uploaded_files = st.file_uploader(
    "上传需求文档与既有用例",
    type=["md", "xlsx"],
    accept_multiple_files=True,
    help="支持多个 .md（需求文档）和 .xlsx（既有测试用例），可混合选择"
)
```

### 10.2 解析

- `.md`：`file.read().decode("utf-8")` 或 `file.read().decode("utf-8-sig")`
- `.xlsx`：复用 `parse_test_cases_file` 或 openpyxl 直接读取

### 10.3 结果分三块

```python
st.subheader("1. 理解内容")
st.markdown(understanding_content)

st.subheader("2. 问题点")
st.markdown(issues_content)

st.subheader("3. 新测试用例表")
st.markdown(cases_markdown)
# Excel 下载
st.download_button("下载 Excel", data=excel_bytes, file_name="测试用例.xlsx", mime="...")
```

### 10.4 仅 Excel 下载

- 使用 `st.download_button`，`data` 为 Excel 二进制（如 `BytesIO` 写入 openpyxl 后 `.getvalue()`）
- 不增加 `output_dir`、Quip、Sheets 等配置项
