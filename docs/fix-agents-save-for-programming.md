# 修复：编辑 Agent 保存/应用问题

> **角色**：@产品  
> **给**：@编程  
> **用途**：按本文档排查并修复「编辑 Agent 后未正确保存/应用」的问题。

---

## 一、现象

用户在「编辑 Agent」页修改 Agent 或 Task 后，点击「保存配置到 config/agents.yaml」，修改未正确生效：
- 保存后再次打开该页，显示仍为旧内容；或
- 下次「生成用例」时，仍使用旧配置。

---

## 二、排查清单（按顺序执行）

| 序号 | 检查项 | 说明 |
|------|--------|------|
| 1 | 文件是否写入 | 点击保存后，检查 `config/agents.yaml` 的 mtime 是否更新；用文本编辑器打开确认内容是否为新值 |
| 2 | 写入路径 | 确认 `AGENTS_CONFIG_PATH`（crew_test.py 中为 `config/agents.yaml`）与实际写入路径一致 |
| 3 | 表单取值时机 | 确认 `a_id`、`a_role` 等来自 `st.text_input` 的当前值，且赋值给 `config` 的时机在 `yaml.dump` 之前 |
| 4 | YAML 序列化 | `yaml.dump(config, ...)` 时，多行字段（如 backstory）是否被正确保留；若格式变化导致解析失败，需调整 dump 参数或改用兼容写法 |
| 5 | 异常被吞掉 | 保存时若有异常，应 `st.error(str(e))` 明确展示，避免静默失败 |
| 6 | 流水线读配置 | `run_pipeline` 是否每次都从 `load_agents_config()` 读取最新文件，而非使用内存中的旧引用 |

---

## 三、必须实现的修复

1. **保存前重新从表单组装 config**  
   不要依赖「遍历时修改原 config」，应在点击保存时，从当前表单控件读取最新值，重新组装 `agents` 和 `tasks` 列表，再写入文件。避免 Streamlit 重跑时取值与预期不符。

2. **保存失败必须提示**  
   写入文件时若发生异常，用 `st.error` 展示具体错误信息，并避免执行 `st.rerun()`，让用户能看到错误。

3. **保存成功后明确反馈**  
   使用 `st.success` 展示「已保存，下次生成用例将使用新配置」，并执行 `st.rerun()` 刷新页面以展示最新内容。

---

## 四、禁止行为

- 不得在未确认写入成功前执行 `st.rerun()`，导致用户看不到错误提示。
- 不得改变 `config/agents.yaml` 的整体结构（agents/tasks 等顶层 key），仅修复保存逻辑。

---

## 五、验收标准

- [ ] 修改任一 Agent 的 role/goal/backstory 后点击保存，`config/agents.yaml` 内容立即更新为新值
- [ ] 刷新页面或再次进入「编辑 Agent」，展示与文件一致
- [ ] 执行一次「生成用例」，流水线使用最新 Agent 配置
- [ ] 保存失败时，页面显示明确错误信息，不静默失败
