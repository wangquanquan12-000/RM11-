## 二、文案配置（ui_texts.yaml）

**目标**：所有新增提示统一从 `ui_texts.yaml` 读取，禁止硬编码。

在 `config/ui_texts.yaml` 中新增或补充：

```yaml
common:
  save_no_change: "内容未变更，无需保存"

agents_tab:
  save_success: "已保存，下次生成用例将使用新配置"

memory_tab:
  save_summary_success: "项目记忆摘要已保存"
  append_success: "已追加到项目记忆"
  append_no_data: "暂无新的运行记录可追加"
  knowledge_refresh_success: "知识库已更新"
  knowledge_refresh_fail: "更新失败：{err}"

run_tab:
  save_success: "已保存到本地"
```

代码中通过 `_get_text(T, "common.save_no_change")`、`_get_text(T, "agents_tab.save_success")` 等读取。

## 三、保存类操作：无变更不写入 + 统一提示

### 3.1 保存 Agent 配置（`_render_module_agents`）

**逻辑**：

1. 用已有或新增的「配置快照」函数（如 `_build_agents_snapshot_from_state`）表示当前表单状态。
2. 对比「当前快照」与「上次已保存快照」：
   - **相同**：
     - 不写 `agents.yaml`；
     - `st.info(_get_text(T, "common.save_no_change") or "内容未变更，无需保存")`；
     - 不 `st.rerun()`。
   - **不同**：
     - 执行写入与二次校验；
     - 成功：`st.success(_get_text(T, "agents_tab.save_success") or "...")`，更新快照、`dirty=False`，`st.rerun()`；
     - 失败：`st.error(...)`（IO / mismatch 等）。

### 3.2 保存项目记忆摘要（`_render_module_memory` 内“项目记忆摘要（高级）”）

**逻辑**：

1. 获取用户输入 `new_summary`。
2. 调用 `load_project_memory()` 得到现有 `old_summary`。
3. 比较 `new_summary.strip()` 与 `old_summary.strip()`：
   - **相同**：
     - `st.info(_get_text(T, "common.save_no_change") or "内容未变更，无需保存")`；
     - 不调用 `update_project_memory`。
   - **不同**：
     - 调用 `update_project_memory` 写入；
     - 成功：`st.success(_get_text(T, "memory_tab.save_summary_success") or "项目记忆摘要已保存")`，必要时 `st.rerun()`。

### 3.3 保存设置（Token/Key/模型）

**逻辑**：

1. 读取 `defaults` / `get_credentials()` 作为 `old_config`。
2. 从表单构造 `new_config`（`quip_token`、`gemini_key`、`gemini_model` 等）。
3. 对比 `old_config` 与 `new_config`：
   - **相同**：
     - `st.info(_get_text(T, "common.save_no_change") or "内容未变更，无需保存")`；
     - 不写文件。
   - **不同**：
     - 写 `defaults.json`；
     - 成功：`st.success(_get_text(T, "run_tab.save_success") or "已保存到本地")`，必要时 `st.rerun()`。

## 四、异步操作：按钮禁用 + loading + 结果反馈

### 4.1 生成用例（`_render_module_quip_to_cases`）

- 点击后：根据 `pipeline_running` 禁用按钮、显示“运行中…”；
- 使用 `st.progress` + `status_text` 展示进度；
- 成功：`st.success(_get_text(T, "run_tab.run_success") or "生成完成")`；
- 失败：`st.error(err_msg)`。

### 4.2 风险报告（`_render_module_risk_report`）

- 点击后：`is_running = True` → 按钮 `disabled=True`，`st.spinner(...)`；
- 成功：`st.success(...)`；
- 失败：`st.error(_get_text(T, "risk_report.timeout_error") or "...")`。

### 4.3 刷新知识库（`_render_module_memory`）

- 已有 spinner；成功/失败文案从 `ui_texts.yaml` 读取；
- 确保无硬编码。

### 4.4 导入类操作（文件夹/单文档/全回归）

- 已有 `st.progress`、`st.spinner`；
- 成功/失败文案统一从 `ui_texts.yaml` 读取。

## 五、「无可追加」场景：从本次运行追加

**逻辑**：

1. 读取 `app_last_run` 或最近一次运行结果；
2. 无运行结果：`st.info(run_first_hint)`；
3. 有运行结果但无可追加内容：`st.info(_get_text(T, "memory_tab.append_no_data") or "暂无新的运行记录可追加")`，不写入；
4. 实际追加成功：`st.success(_get_text(T, "memory_tab.append_success") or "已追加到项目记忆")`。

## 六、验收回归（对应 PRD AC1–AC6）

| 编号 | 验收项 | 验证方式 |
|------|--------|----------|
| AC1 | 保存 Agent 配置时无变更 → 提示「内容未变更，无需保存」，不写入 | 不改表单点保存 |
| AC2 | 保存项目记忆摘要时无变更 → 同上 | 不改摘要点保存 |
| AC3 | 保存设置时无变更 → 同上 | 不改 Token/Key/模型点保存 |
| AC4 | 所有保存/更新成功均有 `st.success` | 逐一点击保存/刷新，确认提示 |
| AC5 | 异步操作按钮禁用 + loading | 生成用例、刷新知识库、导入、风险分析 |
| AC6 | 提示文案全部从 `ui_texts.yaml` 读取 | 全局搜索硬编码中文，确认无遗漏 |
