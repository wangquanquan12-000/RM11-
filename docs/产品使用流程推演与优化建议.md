# 产品使用流程推演与优化建议

> **文档用途**：基于当前代码与 UI 结构推演整体使用流程，识别体验断点与优化机会。
>
> **创建日期**：2025-03-02

---

## 一、当前用户旅程地图（推演）

### 1.1 侧栏与导航结构

```
侧栏
├── 工作台
│   ├── 生成用例（默认首页）
│   ├── 需求风险分析
│   ├── 项目记忆
│   └── 文档问答
└── 高级
    ├── 编辑 Agent
    └── 设置
```

- **首次进入**：`current_page = quip_to_cases`（生成用例）
- **凭证**：来自 `_load_defaults()`（env → credential_store → defaults.json），未保存时需在「配置」展开内填写或跳转设置

---

### 1.2 主流程：生成用例

| 步骤 | 用户操作 | 系统反馈 | 潜在断点 |
|------|----------|----------|----------|
| 1 | 输入需求文档 Quip 链接 | 无即时校验 | 链接格式错误、无权限、文档不存在 → 点「生成」后才报错 |
| 2 | 展开「配置」 | 展示 Token/Key 状态、模型、导出选项 | 新用户不知先填 Token；配置在 expander 内，容易被忽略 |
| 3 | 勾选导出、填 Quip 目标文档 | 无说明 | 「导出到 Quip」未勾时，填 Quip 目标仍会触发追加，逻辑对用户不透明 |
| 4 | 点击「生成测试用例」 | progress 0.2 → spinner → 完成 | 拉取失败、流水线异常、表格解析失败 → 仅部分有 st.error，导出失败静默 |
| 5 | 查看「最近生成」 | 卡片 + Excel/Quip/Sheets 链接 | 无表格时无 Excel、无链接；无失败原因说明 |

**核心路径依赖**：Quip Token + Gemini Key → 拉取 → 四 Agent → 解析表格 → Excel/Quip 导出。任一环节失败，用户常得不到清晰原因。

---

### 1.3 前置准备：项目记忆

| 步骤 | 用户操作 | 系统反馈 | 潜在断点 |
|------|----------|----------|----------|
| 1 | 进入「项目记忆」 | 知识库状态、搜索、导入历史 | 知识库过期自动更新可能失败静默 |
| 2 | 搜索 | 关键词 → 展示匹配条目 | 空搜展示「输入关键词」；新用户不知先导入 |
| 3 | 导入需求（文件夹/单文档/全回归） | 需填 Quip Token（与生成页共用） | Token 未保存时每页都要填；导入流程与生成流程割裂 |
| 4 | 保存摘要 / 从本次运行追加 | st.success / st.info | 从运行追加依赖「先生成一次」，无引导 |

**流程割裂**：新用户若从「生成用例」进，可能从未去过「项目记忆」；Agent 使用的 project_context 来自 memory + agent_knowledge，但用户不清楚「导入」与「生成」的关系。

---

### 1.4 辅助流程：需求风险分析

| 步骤 | 用户操作 | 系统反馈 | 潜在断点 |
|------|----------|----------|----------|
| 1 | 选择文档来源（Quip/粘贴/项目记忆） | 根据来源展示输入区 | 选「项目记忆」时取全量摘要，无「选单篇」能力 |
| 2 | 点击「生成风险报告」 | spinner + 表格/原始输出 | 失败时 st.error；无表格时 st.warning |

---

### 1.5 高级：编辑 Agent

| 步骤 | 用户操作 | 系统反馈 | 潜在断点 |
|------|----------|----------|----------|
| 1 | 编辑 goal/backstory | dirty 标记 | 已按 fix-agents-save-behavior 优化 |
| 2 | 保存 | 成功/无变更/失败 | 二次校验已落地 |
| 3 | 离开页面 | 「有未保存修改」警告 | 符合预期 |

---

### 1.6 设置

| 步骤 | 用户操作 | 系统反馈 | 潜在断点 |
|------|----------|----------|----------|
| 1 | 填 Token/Key/模型 | 无变更检测（若已按交互优化 PRD 落地则有） | - |
| 2 | 保存 | 成功提示 | - |

## 二、体验断点汇总

### 2.1 首次使用门槛高

| 断点 | 现象 | 优化方向 |
|------|------|----------|
| 不知道先配 Token | 直接填链接点生成 → 拉取失败 | 首页空态时突出「请先完成配置」引导，或配置未完成时禁用生成按钮并提示 |
| 不知道「项目记忆」与「生成」的关系 | 直接生成，Agent 无背景 | 在生成页增加简短说明：「导入需求与全回归用例后，Agent 将基于项目记忆生成更贴合业务的用例」 |
| 配置分散 | Token 在设置、配置展开、项目记忆三处可填 | 统一凭证入口，其他页只读「已保存」状态，引导去设置页修改 |

---

### 2.2 主流程反馈不完整

| 断点 | 现象 | 优化方向 |
|------|------|----------|
| 表格解析失败静默 | 无 Excel、无 Quip 链接，但不解释原因 | 按前文方案：`has_tables`、`excel_export_error`、`quip_export_error` 上浮到 UI |
| 导出到 Quip 目标文档的逻辑不透明 | 用户以为只需填地址，但需 `export_quip or export_quip_target` 才触发 | 当 `export_quip_target` 有值时，自动勾选或说明「填写目标文档将自动追加到该文档」 |
| 进度单一 | 只有 0.2 → 1.0，无法感知拉取/Agent 阶段 | 分阶段 progress（拉取 0.2 / Agent 0.5 / 导出 1.0）或 status_text 区分 |

---

### 2.3 项目记忆与主流程割裂

| 断点 | 现象 | 优化方向 |
|------|------|----------|
| 新用户不导入就生成 | Agent 无背景，效果一般 | 生成页空态或首次生成前，弱提示「导入需求与全回归用例可提升生成质量」 |
| 「从本次运行追加」依赖先生成 | 用户先进项目记忆，无运行记录 | 明确文案：「请先在「生成用例」页完成一次生成」 |
| 知识库刷新失败静默 | 自动更新失败不提示 | 失败时 st.warning，带重试入口 |

---

### 2.4 文档问答与风险分析

| 断点 | 现象 | 优化方向 |
|------|------|----------|
| 文档来源「项目记忆」无单篇选择 | 只能全量，长文档可能超 token | 支持「从导入历史中选择单篇」 |
| 风险分析无「上次生成需求」 | 需手动粘贴或拉取 | 增加「使用上次生成的需求文档」快捷入口 |

---

## 三、优化优先级建议

| 优先级 | 优化项 | 影响 | 工作量 |
|--------|--------|------|--------|
| **P0** | 表格解析失败 / 导出失败上浮到 UI | 解决「保存无效」类体验问题 | 小 |
| **P0** | Quip 目标文档：填写时自动视为「导出到 Quip」或明确文案 | 避免用户误解「填了没用」 | 小 |
| **P1** | 首次使用引导：配置未完成时禁用生成 + 引导 | 降低首次失败率 | 小 |
| **P1** | 主流程进度分阶段（拉取 / Agent / 导出） | 缓解长时间等待焦虑 | 小 |
| **P2** | 生成页说明「项目记忆与生成质量的关系」 | 提升导入率 | 小 |
| **P2** | 文档问答 / 风险分析：支持「上次生成需求」快捷入口 | 减少重复输入 | 中 |
| **P3** | 项目记忆「从导入历史选单篇」 | 精细控制文档范围 | 中 |

## 四、典型用户故事与流程优化后对比

### 4.1 故事 A：测试同学首次使用

**当前**：  
进入 → 填链接 → 点生成 → 拉取失败（未配 Token）→ 不知原因 → 放弃或到处找配置

**优化后**：  
进入 → 空态或配置未完成时，生成按钮禁用，文案「请先在「配置」中填写 Quip Token 与 Gemini Key，或前往设置页保存」→ 填完 → 生成成功 → 下载 Excel

---

### 4.2 故事 B：填了 Quip 目标文档但没导出

**当前**：  
填了目标文档地址，未勾选「导出到 Quip」→ 生成完成 → 无 Quip 链接 → 用户困惑

**优化后**：  
当 `export_quip_target` 有值时，自动设置 `do_export_quip=True`（已有逻辑），或在 UI 明确：「填写 Quip 目标文档将自动追加到该文档，无需勾选『导出到 Quip』」

---

### 4.3 故事 C：模型输出非表格，无 Excel

**当前**：  
生成完成 → 无 Excel 下载 → 无任何说明

**优化后**：  
`st.warning("本次未解析到有效的测试用例表格，已跳过 Excel 和 Quip 导出。请检查 Agent 提示词和模型输出格式。")`

---

## 五、小结

当前主流程链路完整，但存在 **首用门槛高、失败反馈不足、配置与流程关系不清晰** 三类问题。  
优先落地 P0：导出失败上浮、Quip 目标文档逻辑透明化；再补 P1 首用引导与进度分阶段，可明显提升可用性与可理解性。
